Compression via the PCA and the SVD

The main topic of this lesson is a data analysis method referred to as Principal Components Analysis (PCA). The method requires computing the eigenvectors of a certain matrix; one way to compute those eigenvectors is to use a special factorization from linear algebra called the Singular Value Decomposition (SVD).

This notebook is simply a collection of notes with a little bit of code to help illustrate the main ideas. It does not have any exercises that you need to submit. However, you should try to understand all the code steps that appear in the subsection entitled, Principal Components Analysis (PCA), as you will need to apply the SVD in a subsequent part of this assignment.

Motivation: data "compression." In previous lessons, we've looked at a few of the major tasks in data analysis: ranking, regression, classification, and clustering. Beyond these, the last problem you'll consider in our class is what we'll call compression.

At a high level, the term compression simply refers to finding any compact representation of the data. Such representations can help us in two ways. First, it can make the data set smaller and therefore faster to process or analyze. Secondly, choosing a clever representation can reveal hidden structure.

As a concrete example, consider the problem of dimensionality reduction: given a dd-dimensional data set, we wish to transform it into a smaller kk-dimensional data set where kdkd.

Choosing the kk dimensions in a clever way might even reveal structure that is hard to see in all dd original dimensions. For instance, look at the examples at the "visualizing PCA" website:

http://setosa.io/ev/principal-component-analysis/

Principal components analysis (PCA)

The method of principal components analysis (PCA) is one such technique. For this example, it would start by viewing these data as four (4) data points, one for each country, embedded in a 17-dimensional space (one dimension per food category). Maximizing projections

If the length of a projected data point is large, then intuitively, we have "preserved" its shape. So let's think of the total length of projections of all the data points as a measure of cost, which we can then try to maximize.

Projection cost. Let J()J() be a cost function that is proportional to the mean squared projections of the data onto :

J()12mi=0m1(x Ti)2.J()12mi=0m1(x^iT)2.

The additional factor of "1/2" is for aesthetic reasons. (It cancels out later on.)

Let's also apply some algebra-fu to the right-hand side to put it into a more concise matrix form:

J()==12T(1mi=0m1x ix Ti)12T(1mXTX)12TC.J()=12T(1mi=0m1x^ix^iT)=12T(1mXTX)12TC.

In the last step, we defined C1mXTXC1mXTX. In statistics, if XX represents mean-centered data, then the matrix CC is also known as the sample covariance matrix of the data.

Principal components via maximizing projections. There are several ways to formulate the PCA problem. Here we consider the one based on maximizing projections.

Start by defining a principal component of the data XX to be a vector, , of unit length that maximizes the sum of squared projections.

To convert this definition into a formal problem, there is a technique known as the method of Langrange multipliers, which may be applied to any minimization or maximization problem that has equality constraints. The idea is to modify the cost function in a certain way that effectively incorporates each constraint: for each constraint you will add to the cost function a term proportional to a dummy parameter times some form of the constraint.

Huh? It's easiest to see this formulation by example. In the case of a principal component, the modified cost function is

J (,)J()+2(1T),J^(,)J()+2(1T),

where the second term captures the constraint: it introduces a dummy optimization parameter, , times the constraint that has unit length, i.e., 22=T=122=T=1, or 1T=01T=0.

The reason to add the constraint in this way should become clear momentarily.

As before, the factor of "1/2" is there solely for aesthetic reasons and will "cancel out," as you'll soon see.

The optimization task is to find the and that maximize J J^:

(,)argmax,J (,).(,)argmax,J^(,).

To solve this optimization problem, you just need to "take derivatives" of J J^ with respect to and , and then set these derivatives to 0. We will like to add that taking gradients and setting it to 0 works here because its a convex optimization problem. The maxima obtained by setting derivatives to 0 is guaranteed to be a global maxima for this problem because of the convex properties of J J^. For problems that are non convex and have a cost function that is harder to differentiate, gradient descent (which you have already seen) is a useful method to find the maxima.

This problem is an eigenproblem, which is the task of computing an eigenvalue and its corresponding eigenvector of C=1mXTXC=1mXTX.

The matrix CC will usually have many eigenvalues and eigenvectors. So which one do you want? Plug the eigenvector back into the original cost function. Then, J()=12TC=2T=2J()=12TC=2T=2. In other words, to maximize J()J() you should pick the with the largest eigenvalue .

Finding an eigenpair via the SVD

So how do you find the eigenvectors of CC? That is, what algorithm will compute them?

One way is to form CC explicitly and then call an off-the-shelf eigensolver. However, forming CC explicitly from the data XX may be costly in time and storage, not to mention possibly less accurate. (Recall the condition number blow-up problem in the case of solving the normal equations.)

Instead, we can turn to the "Swiss Army knife" of linear algebra, which is the singular value decomposition, or SVD. It is an extremely versatile tool for simplifying linear algebra problems. It can also be somewhat expensive to compute accurately, but a lot of scientific and engineering effort has gone into building robust and reasonably efficient SVD algorithms. So let's assume these exist -- and they do in both Numpy and Scipy -- and use them accordingly.

Rank-kk approximations: the truncated SVD

We motivated PCA by asking for a single vector , which effectively projects the data onto a one-dimensional subspace (i.e., a line). You might instead want to represent the original dd-dimensional data points on a kk-dimensional surface or subspace, where ksdksd. As the previous discussion suggests, you could choose the top-kk right singular vectors of XX, v0,,vk1v0,,vk1.

Indeed, there is another "principled" reason for this choice.

Let AmdARmd be any matrix with an SVD given by A=UVTA=UVT. Per the notation above, let smin(m,d)smin(m,d).

Then, define the kk-truncated SVD as follows. Consider any ksks, and let UkUk, kk, and VkVk consist of the singular vectors and values corresponding to the kk largest singular values. That is, UkUk is the first kk columns of UU, VkVk is the first kk columns of VV, and kk is the upper kkkk submatrix of . The kk-truncated SVD is the product UkkVTkUkkVkT.

Now consider the following alternative way to write the SVD:

A=UVT=i=0s1uiivTi.A=UVT=i=0s1uiiviT.

Each term, uiivTiuiiviT is known as a rank-11 product. So the existence of the SVD means that AA may be written as a sum of rank-1 products.

It would be natural to try to approximate AA by truncating the SVD after kk terms, i.e.,

AUkkVTk=i=0k1uiivTi.AUkkVkT=i=0k1uiiviT.

And in fact, there is no rank-kk approximation of AA that is better than this one!

In particular, consider any pair of kk column vectors, YkmkYkRmk and ZkdkZkRdk; their product, YkZkYkZk has rank at most kk. Then there is a theorem that says the smallest difference between AA and the rank-kk product YkZkYkZk, measured in the Frobenius norm, is

minYk,ZkAYkZTk2F=AUkkVTk2F=2k+2k+1+2k+2++2s1.minYk,ZkAYkZkTF2=AUkkVkTF2=k2+k+12+k+22++s12.

In other words, the truncated SVD gives the best rank-kk approximation to AA in the Frobenius norm. Moreover, the error of the approximation is the sum of the squares of all the smallest sksk singular values.

Applied to the covariance matrix, we may conclude that C=1mXTX1mVk2kVTkC=1mXTX1mVkk2VkT is in fact the best rank-kk approximation of CC, which justifies choosing the kk eigenvectors corresponding to the top kk eigenvalues of CC as the principal components.

Summary: The PCA algorithm

Based on the preceding discussion, here is the basic algorithm to compute the PCA, given the data XX and the desired dimension kk of the subspace.

  1. If the data are not already centered, transform them so that they have a mean of 0 in all coordinates, i.e., 1mi=0m1x i=01mi=0m1x^i=0.
  2. Compute the kk-truncated SVD, XUkkVTkXUkkVkT.
  3. Choose v0,v1,,vk1v0,v1,,vk1 to be the principal components.
